# 第二部分：考研择校工具 · 可复用Prompt

## 项目核心Prompt（拿这个直接生成其他学科的择校工具）

```markdown
## 产品定位
我要做一个[学科名称]考研择校工具。它不是一个信息展示网站，而是一个决策辅助工具。
市面上现有的择校产品只是堆砌学校名单和分数线，用户看完还是不知道"我该报哪个"。
我的产品要做的是：让用户输入自己的背景信息，系统自动计算匹配度，告诉TA"冲哪些、稳哪些、保哪些"。

## 核心功能
1. 自我评估页面：用户填写自己的
   - 本科学校层次（985/211/双非一本/二本/专升本）
   - 英语水平（六级多少分/四级多少分/没考）
   - 报考方向（具体专业方向）
   - 擅长题型（论述/材料分析/计算/背诵等）
   - 备考时间（全职/在职/在校）
   - 风险偏好（冲名校/稳妥上岸/无所谓）
   - 意向地区（华东/华南/华北等）

2. 条件筛选页面：
   - 学校层次（985/211/双一流/双非）
   - 地区
   - 给分松紧（水区/旱区/正常）
   - 考试范围（大综合/小综合/单科）
   - 是否保护一志愿
   - 复试淘汰率
   - 冲/稳/保梯度筛选
   - 关键词搜索

3. 匹配评分算法（核心）：
   - 层次匹配（权重30%）：用户背景 vs 学校层次差距
   - 英语适配（权重20%）：英语水平 vs 学校给分松紧
   - 考试范围匹配（权重20%）：擅长题型 vs 考试内容
   - 题型匹配（权重15%）：个人偏好 vs 出题风格
   - 时间匹配（权重15%）：备考时间 vs 竞争程度
   - 风险偏好微调（±5-8分）
   - 输出0-100匹配度 + 冲/稳/保自动分类

4. 数据字段（每个学校需要收集）：
   - 校名、层次、城市、地区
   - 给分松紧、考试范围、作文要求
   - 学科评估排名、招生方向
   - 考试科目、招生人数、竞争程度
   - 是否保护一志愿、复试淘汰率
   - 教授信息、研招网链接、文学院链接
   - 备注（是否公开数据/是否推断）
   - 数据可信度标记（官网确认/经验推断/待核实）

5. 用户体验功能：
   - localStorage保存用户填写的数据
   - CSV导出搜索结果
   - 多校对比弹窗（最多3所）
   - 学校详情弹窗
   - 最佳匹配高亮（≥75分）
   - 深色模式
   - 移动端适配
   - 纠错反馈入口
   - 百度统计接入
   - URL hash分享

## 技术方案
- 单文件HTML，零依赖，纯vanilla JavaScript
- 数据存储在全局数组变量中
- 部署到GitHub Pages
- 百度统计追踪用户行为

## 数据收集策略
- 优先从研招网、各校文学院官网获取公开数据
- 标注数据来源和可信度
- 如果某数据无法确认，标注"待核实"而非编造
- 持续根据用户反馈更新数据
```

## 如何使用这个Prompt

你只需要替换三个地方：
1. **[学科名称]** → 比如"计算机"、"法学"、"医学"
2. **数据字段** → 根据学科特点增删（比如医学加"规培基地"，法学加"法考通过率"）
3. **匹配维度权重** → 根据学科竞争特点调整

然后把这个Prompt发给AI编程工具（Claude/Cursor/Bolt），它就能帮你生成一个完整的择校工具。

## 项目内容运营Prompt（让更多人知道这个工具）

```markdown
## 内容策略
在小红书/知乎/考研群推广这个工具，不发硬广，发以下类型的内容：

1. "我是怎么做这个工具的"——从发现痛点到产品上线的过程
2. "这个工具怎么帮你择校"——使用教程，让用户自己操作
3. "XX学科择校避坑指南"——学科相关的择校知识
4. 工具更新日志——新功能/新数据，让老用户回来

## 关键：不卖工具，卖价值
- 用户关心的不是"这个工具有多少数据"
- 用户关心的是"用了这个工具我能少踩什么坑"
```
